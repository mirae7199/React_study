## Event Handling

이벤트라는 것은 웹 내부에서 발생하는 사용자의 행동<br>
ex) 버튼 클릭, 메세지 입력, 스크롤 등

핸들링은 처리하다라는 것이다.

이벤트 핸들링은 이벤트가 발생했을 때 그것을 처리하는 것을 말한다.<br>
ex) 버튼 클릭시 경고창 노출

버튼을 클릭 시 log에 text를 띄어보자

Button.jsx

```jsx
const Button = (children, text, color) => {
	const onClickButton = () => {
		console.log(text);
		};
	return (
	<button onClick={onClickButton}
//		onClick={() => {
//			console.log(text);
//			}
		style={{color: color}}>
		{text} - {color.toUpperCase()}
		{children}
	</button>
	);
} 
	
```

onClick{ () ⇒ {
  console.log.(text);
}
처럼 익명함수를 사용하여 버튼 클릭 이벤트가 발생했을 때 함수가 호출되게 작성할 수 있다.
익명함수 대신에 객체에 함수를 선언하여 호출할 수 도 있다. onClick = {onClickButton}

## 합성 이벤트 (Synthetic Base Event)

모든 웹 브라우저의 이벤트 객체를 하나로 통일한 형태

웹 브라우저는 종류가 다양하다. 크롬 사파리 파이어폭스 오페라 웨일
그렇기 때문에 웹 브라우저 마다 동작이 조금씩 다르다. 
예를 들어 크롬에서는 타겟이라는 속성이 있는데 사파리에서는 이것을 e타겟이라고 부른다. (예시 일 뿐임) 이렇듯 브라우저 마다 스펙이 달라 발생하는 문제를 Cross Browsing Issue라고 부른다. 이 문제를 편리하게 해결하는 것이 바로 합성 이벤트라는 객체이다. 합성 이벤트 객체는 여러 브라우저들의 규격을 참고해서 하나의 통일된 규격으로 이벤트 객체를 포맷팅 해준다.

SyntheticBaseEvent는 우리가 모든 브라우저에서 사용할 수 있는 통합된 규격의 이벤트 객체다.
